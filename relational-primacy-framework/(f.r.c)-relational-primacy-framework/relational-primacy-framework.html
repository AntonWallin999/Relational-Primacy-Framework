README — RP9 Relational Primacy Framework

Purpose

This repository contains the complete, locked formulation of RP9 — Relational Primacy Framework.

RP9 is a closed relational system built from:

  • a single axiom,
  • a strict causal derivation,
  • and a finite relational function space.

No external assumptions are introduced.
No empirical claims are made.
No symbolic or metaphysical interpretation is required.

This README serves solely as the structural entry point to the framework.

How this framework is organized

The material is intentionally minimal and divided into three non-overlapping layers.
Each layer has a strictly defined role.

Level 1 — Relational Meta Function (Absolute Ground)

This level contains the entire foundation of RP9.

Documents in this layer:

  • establish the single axiom,
  • define the minimal functional terms,
  • and prove the core theorem and its corollary.

Nothing in later layers can modify or extend this level.

If this level is invalid, the entire framework collapses.
If this level holds, everything that follows is necessary.

Level 2 — Relational Derivation (Primary → Tertiary Function)

This level contains the complete relational unfolding of the axiom.

It shows:

  • how the axiom becomes geometric without measurement,
  • how recursion introduces balance,
  • and how fusion of relations produces a closed functional matrix.

No new axioms appear here.
No constants are assumed.
All structures are consequences.

This level completes the internal system.

Level 3 — External Convergence (Non-Foundational)

This level does not add validity to RP9.

It exists to:

  • show convergence with existing practices (e.g. invariance testing),
  • provide a public abstract,
  • and offer a minimal visual comparison.

These documents are illustrative, not evidentiary.

They may be omitted without affecting the framework.

Reading order (strict)

For formal evaluation:

  1. RP9 – Axiom 0
  2. Definition F0
  3. Theorem T0
  4. Corollary C0
  5. Relational Sheet (A)
  6. Relational Sheet (B)
  7. Relational Sheet (C)

All other documents are optional.

What RP9 is not

RP9 is not:

  • a physical theory,
  • a model with tunable parameters,
  • a collection of examples,
  • a symbolic system,
  • or an empirical hypothesis.

RP9 does not explain phenomena.
It defines the minimal conditions under which structure is possible.

Status

  • The framework is structurally complete.
  • The derivation is closed.
  • No further axioms can be added without breaking necessity.

This repository therefore represents RP9 v1.0 (locked).

License and use

The framework is released under CC-BY-NC-SA-4.0.

You may:

  • read,
  • share,
  • reference,
  • and build upon the material,

provided attribution is given and derivative work preserves the same license.

End of README.

APPENDIX — RP9 Relational Primacy Framework

Purpose

This appendix contains supplementary material related to the RP9 framework.

It does not extend, modify, or complete the system.
Its sole purpose is to provide a structured space for derivative, optional, or exploratory material that may arise from the primary framework.

Closed circuit, open system

RP9 establishes a closed relational circuit:

  • the primary relation is fully specified
  • the causal origin is complete
  • no further foundational assumptions are possible

This closure applies only to the origin.

The framework does not define a closed system.

Once the relational cause is given, all further structure, behavior, and manifestation are free to unfold, provided they remain consistent with the primary relation.

RP9 therefore:

  • closes the cause
  • but leaves the consequences open

What belongs in this appendix

Material placed here may include:

  • alternative geometric projections
  • extended relational matrices
  • domain-specific applications
  • visualizations, simulations, or models
  • comparative mappings to other frameworks

All such material is:

  • non-foundational
  • optional
  • context-dependent

None of it is required for the validity of RP9.

What does not belong here

This appendix may not contain:

  • new axioms
  • revisions of Axiom 0
  • redefinitions of relational terms
  • modifications of the relational sheets (A–C)

Any such change would constitute a different framework, not an extension.

Causal responsibility

All material in this appendix carries local causal responsibility.

That is:

  • authors are responsible for their own derivations
  • consequences are not endorsed as necessary
  • validity is domain-specific, not global

RP9 provides the cause.
The appendix records what others do with it.

Status

This appendix is intentionally open-ended.

Its existence does not imply incompleteness of the framework, but rather acknowledges that a closed cause does not imply a closed world.

End of APPENDIX.

Corollary C0 — Secondary Causality for All Non-R0 Structures

Assumptions

  • Axiom R0 is given.

  • Definition F0 is given.

  • Theorem T0 is given.

No further assumptions are introduced.

Corollary

Corollary C0 (Secondary Causality for All Non-R0 Structures)

Every structure, form, operation, or statement that cannot be generated by iterative application of F0 to R0 is necessarily secondary.

Secondary here means: dependent on representation, convention, external parameterization, or added operators, and thus lacking absolute information status in the system's sense.

Direct derivation (ultralogical)

  1. According to Theorem T0, all absolute information is exactly the amount that can be generated from R0 by F0.

  2. Therefore, there are only two classes of structures:

  • Class A: generable by (F0^n(R0)) for some (n).

  • Class B: non-generable by (F0^n(R0)) for some (n).

  1. A structure in Class B cannot be absolute, since it does not arise from the only allowed primary function chain.

  2. For a structure in Class B to still exist, at least one of the following must be true:

  • a new operator has been introduced,

  • an external parameter has been injected,

  • a representation has been replaced by a “thing”,

  • a convention has been chosen.

  1. All of these imply dependence on something not given by R0 and F0.

  2. Thus all Class B is secondary.

The corollary follows necessarily.

Negative verification

If Corollary C0 does not hold, there is a structure that:

  • is not generated by F0 from R0, but

  • is still absolute.

This requires that absolute information exists outside the single absolute generative chain, which contradicts Theorem T0.

Thus C0 must hold.

Status

C0 does not introduce any new content.
It is a pure classification that follows directly from T0:

  • Absolute = generated by (F0^n(R0))

  • Secondary = everything else

Causal chain (locking)

R0 → F0 → T0 → C0 → D0

C0 is now formally a direct consequence, without semantic space.

Definition F0 — Primary relational function (standing fractal)

Definition

F0 (Primary relational function)

Given Axiom R0, F0 is defined as the only allowed operation on the primary relation, such that:

  1. F0 acts exclusively on the relation in R0, not on external objects, representations, or symbols.

  2. F0 introduces no new causal principle, but reuses exactly the same relational conditions that define R0.

  3. F0 is closed under its own application:
    the result of F0 is of the same relational type as its input.

  4. F0 preserves relational identity under iteration up to scale, orientation, and position.

  5. F0 is minimal:
    no simpler non-trivial operation exists that satisfies (1)–(4).

Functional Consequentiality

Repeated application of F0 necessarily generates:

  • recursive relational structures,

  • scale-independent forms,

  • fractal continuity,

without introducing new axioms, definitions, or external assumptions.

Negative Verification

If F0 does not exist:

  • R0 cannot act,

  • no iteration can occur,

  • no structure can arise,

  • and the system remains static and contentless.

If F0 exists but does not satisfy (1)–(5):

  • relational invariance is broken,

  • new causality is introduced,

  • or the iteration collapses.

In all cases, the internal closedness of the system ceases.

Status

F0 is not an axiom.
F0 is not a theorem.

F0 is the necessary primary function that makes R0 work.

This is the standing fractal in the exact sense:
a function that can be applied to its own result without losing identity.

When this definition is inserted:

  • Theorem T0 becomes a direct consequence,

  • Corollary C0 becomes inevitable,

RP9 – Axiom 0

Axiom 0 (Absolute Relational Duality)

Everything that can exist, function or manifest must appear as a relationship between two distinct poles.

This relation is not an object, not a property, and not a magnitude,
without the only natural and primary function that enables stability, causality and structure.

The relationship precedes all form, all energy, all geometry, all dimension and all mathematical representation.
All differentiation, all organization, and all invariance are necessary consequences of this relation when realized as a self-organizing, fractal, geometric function.

This axiom is:

  • non-numeric
  • non-geometric
  • non-temporal
  • and independent of observation

It cannot be reduced to something more fundamental, and cannot be denied without self-contradiction.

Formal Meaning

Axiom 0 states that:

  • A singular, undifferentiated state lacks function and therefore cannot form the basis of any structure.
  • Function requires differentiation.
  • Differentiation requires relationship.
  • Relationship requires at least two poles.

Thus dual relationship is not an assumption, but a necessity for anything to function at all.

Direct consequences (not axioms)

The following are not assumptions, but inevitable consequences of Axiom 0:

  • Stability only occurs through relational balance.
  • Causality arises as direction within relationship.
  • Self-organization occurs when the relationship is iterated.
  • Fractal structure occurs when the same relationship appears scale-independent.
  • Geometry arises as the first stable projection of the relationship.
  • Energy arises as relational rotation.
  • Invariance arises as relational conservation principle.
  • Mathematics arises as a secondary description of relational structure.

Why only one axiom?

Principle justification

An axiom is only legitimate if it cannot be derived from anything else and cannot be further reduced.

Each additional axiom necessarily introduces at least one of the following:

  • a hidden assumption
  • a limitation of generality
  • or an interpretive degree of freedom that violates causal necessity

RP9 therefore uses exactly one axiom, because additional axioms do not strengthen a foundational system — they weaken it.

The single axiom used in RP9 is not a detail, not a constraint, and not a structural preference.
It is the smallest possible functional requirement for anything to exist, function, or differentiate at all.

This axiom operates at the root level, prior to geometry, energy, dimension, or mathematics.
It is minimal in content, yet sufficient in capacity to recursively generate the entire relational and geometric structure of reality as a consequence of the same underlying relation.

All higher-order structures — including geometry, fractal scaling, invariance, and mathematical representation — arise not as assumptions, but as necessary consequences of this single relational function.

RP9 therefore contains:

  • one axiom
  • zero free parameters
  • and no auxiliary postulates

Everything else is derivation.

Logical Necessity

If more than one axiom is required, it follows that:

  • the system is not causally closed
  • or that certain consequences are not necessary

RP9's goal is not to postulate structure, but to show what must follow if function is to be possible at all.

Axiom 0 is the only statement that:

  • cannot be derived
  • cannot be avoided
  • and cannot be replaced

Everything else in RP9 is deduction, consequence or presentation.

Epistemic Discipline

RP9 strictly distinguishes between:

  • Axiom – that which cannot be otherwise
  • Consequence – that which must follow
  • Presentation – how consequences are shown, modeled or visualized

Energy models, rotation models, dimensional transitions and mathematical formalisms are therefore validating projections, not supporting foundations.

The validity of the system rests solely on Axiom 0.

Final Lock

RP9 is a system with:

  • an axiom
  • zero free parameters
  • and full causal closure

All complexity in RP9 is emergent, not assumed.

Theorem T0 — Absolut informationsgrund

Förutsättningar

  • Axiom R0 är givet.
  • Definition F0 (Primär relationell funktion) är givet.

Inga ytterligare axiom, operatorer eller representationer införs.

Sats

Theorem T0 (Absolut informationsgrund)

All absolut information i systemet är nödvändigt och uteslutande härledbar som en konsekvens av R0 genom upprepad tillämpning av F0.

Varje påstående, struktur eller form som inte kan genereras genom iterativ tillämpning av F0 på R0 saknar absolut informationsstatus.

Direkt härledning (ultralogisk)

  1. Enligt Axiom R0 existerar exakt en minimal, stabil primär relation.

  2. Enligt Definition F0 existerar exakt en tillåten funktion som:

    • verkar på R0,
    • bevarar relationell identitet,
    • och är sluten under iteration.

  3. Därmed är alla relationella strukturer som:

    • uppstår genom iteration av F0,
    • av samma relationella typ som R0,
      absoluta i systemets mening.

  4. Varje struktur som:

    • kräver ytterligare operatorer,
    • introducerar ny kausalitet,
    • eller beror på representation,
      kan inte vara absolut.

Satsen följer nödvändigt.

Negativ verifikation

Om Theorem T0 inte gäller:

  • måste antingen en absolut struktur existera utan att vara genererad av F0,
  • eller en absolut struktur existera utan grund i R0.

Båda fallen strider mot:

  • minimaliteten i R0, och
  • slutenheten i F0.

Alltså är T0 oundvikligt givet R0 och F0.

Status

Theorem T0 är inte ett oberoende resultat.
Det är den omedelbara och nödvändiga konsekvensen av att:

  • relationen (R0) är primär, och
  • funktionen (F0) är sluten och invariant.

Kausalkedjan (nu korrekt låst)

Relation (R0)
→ Funktion (F0)
→ Absolut informationsgrund (T0)
→ Sekundär kausalitet (C0)
→ Stängning (D0)

Nu finns ingen logisk lucka mellan R0 och T0.
Absoluthet uppstår inte som antagande, utan som verksam konsekvens.
